Workflow case study

agentic-ds in practice

WC26 Pulse showcases agentic-ds, a guided, warrant-first workflow for AI-assisted data analysis. The skill treats agents as fast producers, not automatic authorities: every important number must be scripted, every claim must survive a challenge, and every gate that changes the story needs human judgment.

Skill sources

This description is grounded in the agentic-ds skill files: SKILL.md, references/workflow.md, and references/templates.md. Those files define the workflow as a zero error, warrant-first operating system for data analysis with AI agents.

The loop

  1. Read the state. Start from STATUS.md, CHECKLIST.md, the decision log, and the latest session log so the agent and human re-enter the same project.
  2. Check for rot. Use freshness and drift checks to surface stale, missing, orphaned, or unexplained outputs before citing anything.
  3. Orient the user. Summarize what happened since the last visit, what changed, and what needs judgment, assuming everyone has forgotten the details.
  4. Nudge honestly. Call out stale outputs, unexplained drift, unfiled results, and unchecked gates instead of hiding them behind progress.
  5. Propose one next step. The checklist is a gate sequence, not a menu. The agent advances the current gate unless the human redirects.
  6. File as work happens. Findings, decisions, and caveats are written to files with provenance while they happen, not reconstructed later.
  7. Close with memory. End by writing the session log, updating the checklist, regenerating status, and checking that the files match the project story.

The six habits

Decide first

Write the question, metric, population, comparison, baseline, and kill condition before the analysis runs.

Script numbers

No important number lives only in chat or a notebook. Evidence comes from numbered scripts and saved outputs.

File results

A finding or decision is real only when it is recorded with provenance. Conversation is not durable memory.

Challenge claims

No claim is supported until it passes a pre-named test that could have killed it.

Open and close

Every session starts from project state and ends by updating the shared state.

Keep human gates

The agent drafts, computes, and checks. The human signs specs, decisions, supported claims, and final framing.

Why this matters here

Reproducible

A stranger can follow the trail from public data to scripts to tables to site JSON. The prediction scoreboard is regenerated from committed data, not manually edited.

Transparent

The project tells visitors when a model failed, what replaced it, where public data lags, and which outputs are experimental.

Auditable

The claims register, findings, decision log, source traces, session logs, and status dashboard let a reviewer inspect the project mid-flight.

What about Claude and Codex?

Working with more than one AI agent is compatible with agentic-ds because the workflow does not trust agent memory. It trusts filed artifacts. Claude, Codex, or any future agent can contribute code and copy, but the project state lives in the brief, spec, decisions, claims, findings, scripts, and dashboard.

The result

WC26 Pulse is not just a prediction site. It is a public demonstration of warrant-first data science: decide before analyzing, script every number, challenge every claim, keep failures visible, and let the human sign the gates.
